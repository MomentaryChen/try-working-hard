# try-working-hard

**語言：** [English](README.md) · 正體中文（本頁）

在鍵盤與滑鼠皆無輸入達您設定的時間後，才讓游標微動再移回（Windows 以系統閒置時間為準）。可選**路徑模式**（直線、圓形、方形）或**自然模式**（不規則微移，可選低機率點擊與捲動），並以 GUI 設定間隔與選項。僅供個人合法用途（例如避免簡報或閱讀時螢幕休眠），請遵守公司與服務條款。

## 環境需求

- **Windows**（使用 Win32 API 移動游標）
- **Python** 3.10 或以上
- 建議使用 **[uv](https://docs.astral.sh/uv/)** 管理環境與執行

## 安裝與執行

在專案根目錄：

```powershell
uv sync
uv run try-working-hard
```

或使用模組方式：

```powershell
uv run python -m mouse_jiggler
```

### Windows 單檔執行檔（不需安裝 Python）

在 **GitHub Releases** 會提供含版本號的單一執行檔：`try-working-hard-vX.Y.Z.exe`（例如 `try-working-hard-v1.2.0.exe`，PyInstaller 打包、無主控台視窗）。請到對應版本的 **Assets** 下載後直接執行。

- 自行或 CI 如何建置：見 [docs/WINDOWS-BUILD.md](docs/WINDOWS-BUILD.md)（含 SmartScreen／防毒說明，以及 Briefcase 等替代方案簡介）。
- **發佈方式：** 推送 `v*` 標籤（例如 `v1.0.0`），[`.github/workflows/release.yml`](.github/workflows/release.yml) 會跑測試、建置並上傳至該 GitHub Release。

### 鍵盤與無障礙

- **F1** 開啟說明；**F2 / F3 / F4** 切換主要頁面；在**首頁 → 控制面板**時 **F5** 開始、**Shift+F5** 停止（條件允許時）；**Enter**／**Esc** 在**主視窗顯示中**時分別同開始／停止（僅在系統匣、主視窗未顯示時不作用）；**F6** 切換控制／紀錄。可**點擊「間隔」「間隔隨機」「位移」「路徑速度」「活動模式」「移動路徑」標籤**將焦點移到欄位；間隔欄位下的 **30s / 1m / 5m / 10m** 為快速設定。**Tab / Shift+Tab** 切換焦點。
- CustomTkinter 多數元件以**畫布**繪製，**螢幕閱讀器**支援度與完整原生 Win32 不同。詳見 [docs/ACCESSIBILITY.md](docs/ACCESSIBILITY.md)。

## 使用方式

### 啟動提示流程

- 一般啟動（非 `--start-in-tray`）時，程式會先顯示自訂樣式的啟動提示視窗（非系統預設訊息框）。
- 語言改為在啟動提示視窗先選（**繁中 / English**），且提示文案會立即切換。
- 選 **開啟流程簡介** 會進入可切換 **上一步 / 下一步 / 完成** 的分步驟 UI 教學。
- 教學視窗新增「**主要功能**」摘要區與視覺化「**進度指示**」（進度條 + 步驟/百分比）。
- 「不再顯示」會記錄到本機 `config.json` 的 `intro_acknowledged`。

1. 主視窗啟動時**預設最大化**；可從標題列還原或調整大小。
2. 每次一般啟動（非 `--start-in-tray`）會先顯示一次**用途提醒**對話框；請僅在合法且符合公司／學校／服務規範的情境下繼續使用。
3. 在 **設定** 頁，標題固定於上方，選項集中在卡片內；視窗較矮時可在**卡片內捲動**以看到全部選項。選擇 **外觀**（**深色**／**淺色**；`ui_theme`，預設 **light**），並以 **繁中 / English** 切換語言。排程可啟用後使用 **開始**／**結束**（24 小時制 **HH:MM**，結束不含）作為相容設定，同時 `config.json` 已支援進階規則：`schedule_window_segments_text` 可設定多段（例如 `09:00-12:00,13:00-18:00,21:00-23:00`）、`schedule_include_weekends` 可控制是否含週末、`schedule_cron_text` 可填入 cron-like（5 欄位，多條用 `;` 分隔）。既有 `schedule_window_start_text` / `schedule_window_end_text` 仍可使用。**Windows** 且有系統匣時可 **隨 Windows 啟動** 與 **關閉視窗時縮到系統匣**（見後述）。「**關於與更新**」區塊可查看目前版本、開啟聯絡/回報頁、手動檢查新版本，並可切換**啟動時自動檢查更新**（`auto_check_updates`）。啟用排程時，**首頁**狀態列**下方**會顯示規則摘要。
4. 在「間隔」輸入數字（鍵盤／滑鼠無輸入須累積達此時間後才微動），並以 **分鐘／秒** 切換單位（分鐘可為小數，例如 `0.5` 約 30 秒；與以秒為單位時的「最小可設秒數」同樣有下限）。欄位旁或下方有 **30s / 1m / 5m / 10m** 快速鈕可一鍵設定。
5. 可選：**間隔隨機（± 秒）**（`0` ＝ 關閉）：每次所需的閒置間隔在 **間隔 ± N 秒** 之間均勻隨機（下限與主間隔相同）。範圍 **0–3600**，存於 `config.json` 的 `interval_jitter_text`。
6. 在「位移（像素）」輸入 **0–500** 的整數。**路徑模式**：直線＝水平距離、圓＝半徑、方＝邊長。**自然模式**：游標偏離起點的最大距離。**0** 表示該次不位移。預設 **100**。
7. 在「**路徑速度**」輸入 **1–10** 的整數：每次微動演進的快慢（數字愈大愈快）。預設 **5**，存於 `config.json` 的 `path_speed_text`。
8. 選擇 **活動模式**：**路徑模式** 搭配 **直線／圓形／方形**（`motion_pattern`）。**自然模式** 為不規則微移，可勾選**偶發左鍵**與**偶發滾輪**（每次觸發機率低；還原位置後送出）。偏好鍵 `activity_style`（`pattern`／`natural`）、`natural_rare_click`、`natural_rare_scroll`。視窗高度不足時，首頁控制區可**捲動**。
9. 按 **開始** 啟動定時微動；**停止** 可結束。在左側欄選 **首頁**，再以 **控制面板 / 紀錄** 分段切換（英文模式下按鈕文字為英文）。
10. 執行中 **首頁頂端狀態區** 顯示**下次可能微動**的 **倒數**（分:秒；滿一小時以上則為 時:分:秒），依閒置時間與間隔計算（持續閒置時，兩次微動之間至少隔一個完整間隔）與**顏色提示**；若啟用上班時段，暫停時顯示排程相關倒數；停止時為中性樣式。
11. **預設**：關閉視窗會 **停止排程並結束程式**。若勾選「關閉視窗時縮到系統匣」，關閉後會隱藏視窗並在通知區保留圖示，**排程會繼續**；在圖示上右鍵選單文字會依目前語言顯示。亦可用 `uv run python -m mouse_jiggler --start-in-tray` 或 打包之 **.exe** 加 **`--start-in-tray`** 直接以**僅系統匣**啟動（不先顯示主視窗）。
12. 在 **設定** 頁可使用 **開啟設定檔**，以系統預設程式開啟 `config.json`（Windows 通常位於 `%APPDATA%\try-working-hard\`；未設定 `APPDATA` 時為 `~/.try-working-hard/`）。若檔案尚未存在，會先依目前設定寫入再開啟。
13. 左側 **分析** 頁顯示 **Matplotlib** 統計圖（觸發次數、每日運作時長、路徑比例含**自然模式**），下方文字區與 **首頁「紀錄」** 同步唯讀顯示。統計資料另存為 **`analytics.json`**，與 **`config.json`** 位於**同一資料夾**。

## 技術說明

- GUI：**CustomTkinter**—**淺色**改為更乾淨的 SaaS 色票（背景 **#F5F7FB**、側欄/卡片 **#FFFFFF**、邊框 **#E2E8F0**），主色互動為 **#3B82F6 / #2563EB / #1D4ED8**；**深色**改為現代深色系（背景 **#1E1E2E**、側欄 **#181825**、卡片 **#24273A**、邊框 **#313244**），主色互動為 **#89B4FA / #74C7EC / #B4BEFE**。**首頁**內以分段切換 **控制面板 / 紀錄**。
- **分析**：以 **Matplotlib**（TkAgg）嵌入；統計資料存於 **`analytics.json`**（與 **`config.json`** 同目錄）。
- 滑鼠：**ctypes** 呼叫 `user32.GetCursorPos` / `SetCursorPos`；自然模式可額外以 `mouse_event` 送左鍵與滾輪；排程以 `GetLastInputInfo` 與 `GetTickCount` 取得閒置時間
- 系統匣：**pystray**、圖示繪製：**Pillow**（主視窗與匣區共用 PNG；重建 `.exe` 時另以 [`packaging/app.ico`](packaging/app.ico) 作為檔案圖示，見 [docs/WINDOWS-BUILD.md](docs/WINDOWS-BUILD.md)）
